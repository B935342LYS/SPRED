# 1.42 Game Mode Pitch Visualization

## 1. Scope

이번 작업은 노래방 / 게임 모드의 1차 UI skeleton과 2차 pitch visualization 일부를 연결한 것이다.
점수 산정에 들어가기 전에 practice mode 진입, 마이크 음정 인식, 초록색 pitch dot 표시, 기본 입력 잠금 정책을 먼저 검증할 수 있게 만들었다.

구현 범위:

- player 영역 우하단의 `practice mode` toggle 버튼
- player / practice panel 수평 분할 UI
- ready / countdown / playing / paused / finished 중심의 게임 모드 런타임 상태
- practice playback 시작 전 `3, 2, 1` countdown과 beep
- practice mode 중 edit / seek / loop / layout / file 계열 조작 잠금
- practice ready/stopped 상태에서 active track layer toggle 허용
- `pitchy` 기반 마이크 pitch detection runtime
- clarity / RMS / layout frequency range 최소 필터
- `GamePitchFrame` runtime state
- score canvas 위 형광초록 pitch dot 표시
- 현재 active target 후보 기반 pitch dot 보정
- pitch class 보정은 옥타브 관계로 볼 수 있는 입력에만 적용
- target 후보 조회 시점은 playback 기준선보다 약 `100ms` 앞선 score time 사용
- `test:game` pitch math 단위 테스트

아직 구현하지 않은 범위:

- `GameJudgeTarget` 생성
- `Perfect / Ok / Bad / Miss` scoring sample 판정
- score time 기준 점수 누적
- articulation count bonus

## 2. Module Flow

흐름은 다음과 같다.

```text
getUserMedia MediaStream
  -> createGamePitchInputRuntime()
  -> AnalyserNode.getFloatTimeDomainData()
  -> PitchDetector.findPitch()
  -> clarity / RMS / frequency range filter
  -> GamePitchFrame
  -> syncGamePitchOverlay()
  -> active track NoteEvent 후보 조회
  -> pitch class octave-only 보정
  -> renderer layout y 좌표 변환
```

`GamePitchFrame`은 score JSON에 저장하지 않고 `AppState.gameMode`의 런타임 상태에만 둔다.

## 3. Implementation Notes

- `src/app/game/game_pitch_input.ts`
  - 마이크 stream, `AudioContext`, `AnalyserNode`, `PitchDetector` lifecycle을 관리한다.
  - `requestAnimationFrame`마다 pitch frame을 만들고, practice mode 종료 시 `dispose()`로 AudioContext와 RAF를 정리한다.
- `src/app/game/game_pitch_math.ts`
  - frequency to MIDI / cent 변환, RMS 계산, layout 기반 frequency range 계산을 담당한다.
  - `resolveClosestPitchClassCandidateMidi()`는 target 후보와 pitch class 오차가 `100 cent` 미만일 때만 옥타브 보정을 적용한다.
  - target이 C인데 입력이 D나 C#처럼 다른 계이름으로 튀면 target 주변으로 접지 않고 원 입력 pitch를 유지한다.
- `src/app/game/game_pitch_overlay.ts`
  - 최신 voiced pitch frame을 현재 renderer layout의 note row y 좌표로 변환한다.
  - dot의 x 좌표는 현재 score viewport 왼쪽 기준선 근처에 고정한다.
  - 현재 viewport 왼쪽 playback 기준선 tick을 tempo map으로 seconds에 변환하고, `+100ms`를 더한 뒤 다시 tick으로 변환해 target 후보 조회 시점으로 사용한다.
  - `TickTimeMapper`는 pitch frame마다 새로 만들지 않고 `timingTimeline` 참조 기준으로 cache한다.
- `src/app/game/game_types.ts`
  - `isGameModeLocked()`는 edit/seek/file/layout처럼 게임 세션 전체에서 금지할 조작에 사용한다.
  - `isGameModeTrackChangeLocked()`는 active track 변경 전용으로, practice `ready` 상태에서는 track toggle을 허용한다.
- `src/app/app_track_binding.ts`
  - practice `ready` 상태에서는 active track toggle을 허용한다.
  - countdown / playing / paused / finished / preparing 상태에서는 active track 변경을 막는다.

## 4. Source Note

- `pitchy`
  - URL: `https://github.com/ianprime0509/pitchy`
  - npm package: `pitchy@4.1.0`
  - license: MIT
  - usage: direct dependency
  - used API: `PitchDetector.forFloat32Array()` and `findPitch()`
  - role: real-time pitch detection and clarity value provider

Pitchy는 noise reduction을 완전히 대신하지 않는다.
따라서 앱에서 직접 RMS threshold, clarity threshold, layout frequency range filter를 적용한다.

## 5. Verification

최근 확인한 명령:

```text
npm run test:game
npm run test:track
npm run typecheck
npm run build
git diff --check
```

검증 내용:

- `test:game`
  - A4 440Hz -> MIDI 69 변환
  - RMS 계산
  - layout 기반 frequency range 계산
  - C5 입력을 C4 target 주변으로 접는 옥타브 보정
  - 다른 계이름 / 반음 튐 입력은 target 주변으로 접지 않는 정책
  - practice ready 상태의 active track 변경 허용, playing/paused 상태의 차단
- `test:track`
  - active track toggle의 빈 목록 허용
  - inactive track alpha 표시
  - active track 기반 audio schedule filtering
  - active track 변경 시 renderer input 갱신

브라우저 수동 검증이 필요한 항목:

- 실제 마이크 권한 허용 / 거부 흐름
- 무음 상태에서 pitch dot 숨김
- 안정적인 입력음에서 pitch dot row 위치
- pitch가 다른 계이름으로 튀는 상황에서 target 보정이 과도하게 걸리지 않는지 여부
- practice ready 상태에서 track toggle 후 playback target이 바뀌는지 여부
