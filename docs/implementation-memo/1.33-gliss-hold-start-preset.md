# 1.33. Gliss hold-start preset

## 1. 목적

이 메모는 Gliss 패널에 추가한 `- + start` 프리셋 입력을 기록한다.

이 기능은 저장 형식, parser, analyzer 계약을 바꾸지 않는다.
기존 note cell rawText 조합을 edit UI에서 빠르게 생성하는 shortcut이다.

## 2. 배경

글리산도 연결 시작점을 만들 때 사용자는 자주 다음 조합을 입력한다.

```text
-@g(a,S)
```

기존 UI에서는 이 값을 만들기 위해 다음 패널을 모두 조작해야 했다.

- Default 패널에서 defaultText를 비우거나 의도적으로 제외
- Long 패널에서 `-` 선택
- Gliss 패널에서 `start`와 id 선택

start anchor를 여러 개 입력한 뒤 일반 note 입력으로 돌아갈 때도 세 패널 상태를 다시 되돌려야 해서 반복 작업 비용이 컸다.

## 3. 구현 정책

Gliss kind 드롭다운에 `- + start` 항목을 추가했다.

내부 값:

```text
holdStart
```

합성 결과:

```text
-@g(id,S)
```

정책:

- `- + start`가 선택된 동안 계속 같은 프리셋 입력으로 동작한다.
- Gliss id select 값은 그대로 반영한다.
- Default 패널 값은 UI 상태로 유지하지만 rawText 합성에는 반영하지 않는다.
- Long 패널 값도 UI 상태로 유지하지만 rawText 합성에는 반영하지 않는다.
- Tremolo, absolutePitch, microPitch 값도 이 프리셋 rawText에는 반영하지 않는다.
- current rawText preview에는 실제 적용될 `-@g(id,S)`가 표시된다.

## 4. 경계 해석

이 기능은 Gliss 패널이 Long 패널의 `-` 의미를 소유한다는 뜻이 아니다.

경계는 다음처럼 해석한다.

- Long 패널의 `-`는 일반 hold 입력이다.
- Gliss 패널의 `S/M/E`는 gliss anchor 입력이다.
- `- + start`는 두 토큰을 자주 함께 쓰는 작업을 위한 edit UI shortcut이다.

따라서 parser/analyzer에는 새 token이 생기지 않는다.
저장되는 값도 기존 parser가 이미 처리하는 `-@g(id,S)`이다.

- `regression-code/index.html`

- `regression-code/src/app/edit/edit_default.ts`
  - `GlissEditInput.kind`에 `holdStart` 추가

- `regression-code/dev/test_edit.ts`
  - Default/Long/Trem/Pitch 값이 있어도 `holdStart`가 `-@g(id,S)`만 만드는지 검증

확인 명령:

```text
npm run typecheck
npm run test:edit
npm run test:parse
npm run test:analyze
npm run build
```

브라우저 수동 확인 항목:

- Gliss kind에서 `- + start` 선택
- id 변경 시 preview가 `-@g(id,S)`로 바뀌는지 확인

- Long/Trem/Pitch 패널 값이 남아 있어도 프리셋 입력에 섞이지 않는지 확인
