# 1.24. Step5 Global Timeline Visualization

## 1. Purpose

This memo fixes the first renderer rules for global timeline visualization after audio dynamics automation was connected.
It supplements the active renderer and analyzer specs without changing parser/analyzer storage contracts.

## 2. Input Boundary

Renderer conversion consumes analyzer output only.

```text
AnalysisResult.timingTimeline
AnalysisResult.dynamicsTimeline
-> CanvasMarkerItem
-> marker layer draw
```

The renderer does not reparse global row `rawText`.
Raw global text remains a separate `CanvasGlobalTextRenderItem` path.

## 3. BPM Change Markers

BPM change markers are vertical lines over the score area.

- Initial `col = 0` BPM state is not drawn as a change marker.
- Instant BPM changes are drawn as translucent sky-blue vertical lines.
- Linear BPM ramps are drawn at the ramp start.
- Linear BPM ramp end points are also drawn as translucent sky-blue vertical lines.
- Increasing BPM ramps are red.
- Decreasing BPM ramps are green.
- `beatsPerBar` and `stepsPerBeat` changes do not create BPM change markers.
- If a linear BPM ramp is split by a beat/step segment boundary, only the first segment of the continuing ramp draws the marker.
- Only one BPM marker is drawn per tick.
- If a ramp end and a new ramp start share the same column, the new ramp start direction marker takes precedence over the sky-blue ramp end marker.
- If `><` starts a new ramp in the same direction as the previous ramp, it still draws one direction marker at that column.

## 4. Dynamics Guide

Dynamics guide items are drawn on the dynamics global row.

- Value range follows the active global parser: `0..150`.
- `150` fills the full dynamics row height.
- `0` is invisible.
- Thickness is proportional to row height.

```text
thickness = rowHeight * (value / 150)
```

- Instant segments draw a constant-thickness light gray horizontal band.
- Linear segments draw a trapezoid whose start and end thickness match the interpolated dynamics values.
- The band is vertically centered in the dynamics row.
- Global rawText remains drawn above the guide through the existing global text layer.

## 5. Color Policy

Initial colors:

```text
BPM instant: rgba(96, 200, 255, 0.45)
BPM accel:   rgba(255, 80, 80, 0.45)
BPM rit:     rgba(80, 220, 130, 0.45)
Dynamics:    rgba(220, 220, 220, 0.45)
```

## 6. Implementation Notes

- Add global timeline marker variants to `CanvasMarkerItem`.
- Build BPM markers from `timingTimeline` by comparing adjacent BPM segment continuity.
- Build dynamics guide markers from `dynamicsTimeline` and the segment source row.
- Draw dynamics guides before beat/bar/BPM/gliss markers so text and foreground markers stay legible.
