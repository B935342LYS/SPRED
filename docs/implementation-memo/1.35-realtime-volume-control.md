# 1.35 Realtime Volume Control

## Context

Firefox tester reported that the volume control did not appear to work.

The previous UI binding listened only to the range input `change` event and recreated the playback backend when volume changed. Browser range input commit timing can differ, and recreating the backend can make volume changes feel delayed or interrupted.

## Change

`AudioBackend` now exposes `setMasterVolume(volume)`.

`createOscillatorBackend()` keeps one master `GainNode` for final output volume and updates it directly. Note and gliss envelopes now use event velocity only, while the master gain handles global volume.

The previous output path effectively multiplied the UI volume twice. To avoid a sudden loudness jump after moving volume control into the master gain, the backend converts UI volume to `volume * volume` for the master gain.

The playback UI listens to both `input` and `change` on the volume slider. `input` updates the playback backend and note-preview backend immediately without rebuilding playback runtime.

Waveform changes still rebuild the backend because oscillator type affects newly created oscillator nodes.

## Scope

- Web Audio remains the only audio backend.
- YouTube iframe volume is not controlled by this slider.
- Playback position preservation logic is unchanged for waveform changes and score rebuilds.
