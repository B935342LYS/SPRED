# 1.40 Undo / Redo History Implementation

## 1. Scope

이번 작업은 edit mode의 note/global cell rawText 변경을 되돌리기 위한 Undo / Redo 1차 구현이다.

포함 범위:

- 단일 note/global cell edit
- range delete
- range paste
- drag edit transaction

제외 범위:

- score file load
- Details metadata edit
- YouTube metadata reload
- Clear All
- Expand Right / Trim Right
- layout apply / layout preset apply
- local save / file download
- track/view/playback runtime state

## 3. Data Model

첫 구현은 `ScoreFile` 전체 snapshot을 저장하지 않고 cell patch만 저장한다.

```ts
type CellHistoryPatch = {
  selection: ScoreEditSelection;
  beforeRawText: string | null;
  afterRawText: string | null;
};
```

`null`은 해당 cell이 존재하지 않았거나 삭제되었음을 뜻한다.
undo 적용 시 `beforeRawText`, redo 적용 시 `afterRawText`를 기존 `ScoreTextEdit[]` batch로 변환한다.
이 덕분에 undo/redo도 기존 partial rebuild 경로를 그대로 사용한다.

## 4. History Limit

기본 history 한도는 50 entry이다.

이 값은 cell 수가 아니라 사용자 조작 단위 기준이다.
cell patch 방식이므로 메모리 비용은 전체 악보 크기가 아니라 변경된 cell 수와 rawText 길이에 비례한다.
`rawText`는 100자 제한이 있으므로 일반 edit에서는 비용이 작다.

## 5. Drag Transaction

처음 구현에서는 drag edit이 RAF batch마다 history entry를 만들 수 있었다.
후속 수정으로 pointer drag가 실제 시작된 순간 transaction을 열고, pointerup/pointercancel에서 한 번만 history entry를 push하도록 바꿨다.

렌더링과 partial rebuild는 드래그 중에도 기존처럼 batch마다 즉시 수행한다.
history 기록만 transaction 종료 시점까지 지연한다.

## 6. UI / Shortcut

연결된 UI:

- Undo button
- Redo button

연결된 shortcut:

- `Ctrl+Z` / `Cmd+Z`
- `Ctrl+Y`
- `Ctrl+Shift+Z` / `Cmd+Shift+Z`

input, textarea, select, contenteditable 안에서는 브라우저 기본 편집 undo/redo를 방해하지 않는다.
