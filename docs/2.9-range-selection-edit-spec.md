# 2.9. Range Selection / Bulk Edit 명세

## 1. 문서 목적

본 문서는 `회귀 코드`의 edit mode에서 Ctrl + drag 영역 선택과 이후 bulk delete / copy / paste 기능을 정의한다.

목표는 다음이다.

- 기존 drag edit 입력과 충돌하지 않는 Ctrl + drag 영역 선택 gesture를 추가한다.
- 선택 영역을 score JSON에 저장하지 않는 runtime UI 상태로 관리한다.
- 선택 영역 삭제는 기존 score text edit batch / partial rebuild 경로를 재사용한다.
- 복사/붙여넣기는 저장 포맷을 바꾸지 않고 같은 layout 안에서 먼저 동작하게 한다.
- 복사 이후 paste preview를 표시해 붙여넣기 column을 사용자가 예측할 수 있게 한다.

본 문서는 `docs/2.2-ui-state-edit-mode-spec.md`, `docs/2.6-track-layer-ui-spec.md`, `docs/2.8-edit-invalidation-and-partial-rebuild-spec.md`의 후속 구현 명세이다.

## 2. 기본 원칙

Range selection은 score 저장 데이터가 아니다.

```text
pointer input
  -> hit test
  -> range selection runtime state
  -> selection overlay render
  -> optional bulk command
  -> existing score text edit batch
```

영역 선택은 parser, analyzer, audio 계약을 바꾸지 않는다.
bulk delete / paste가 실제 score cell을 바꿀 때만 기존 edit mutation 경로를 사용한다.

첫 구현은 desktop edit mode만 대상으로 한다.
mobile edit mode는 기존 프로젝트 결정에 따라 초기 범위 밖이다.

## 3. 상태 모델

`AppState`에는 단일 cell 선택과 별도로 range selection 상태를 둔다.

```ts
type ScoreRangeSelection = {
  rowKind: "note" | "global";
  startRowId: RowId;
  endRowId: RowId;
  rowIds: RowId[];
  startCol: number;
  endColExclusive: number;
  trackIds: TrackId[];
};
```

규칙:

- `rowKind`는 `"note"` 또는 `"global"`만 허용한다.
- `gap` row는 range selection 대상에서 제외한다.
- `rowIds`는 현재 renderer row order 기준으로 start/end 사이에 포함된 같은 `rowKind` 행 목록이다.
- `startCol <= col < endColExclusive`를 선택 범위로 본다.
- note range의 `trackIds`는 선택 확정 시점의 `activeTrackIds`를 사용한다.
- global range의 `trackIds`는 빈 배열로 둔다.

초기 구현에서는 `selection`과 `rangeSelection`을 동시에 둘 수 있다.
단일 click은 `selection`을 갱신하고 `rangeSelection`을 해제한다.
Ctrl + drag는 `rangeSelection`을 갱신하고 단일 `selection`은 마지막 anchor 또는 기존 값을 유지할 수 있다.

권장 AppState 확장:

```ts
type AppState = {
  selection: ScoreSelection | null;
  rangeSelection: ScoreRangeSelection | null;
  rangeClipboard: ScoreRangeClipboard | null;
  pastePreview: PastePreviewState;
};
```

Paste preview 상태:

```ts
type PastePreviewState = {
  anchorCol: number | null;
};
```

규칙:

- `rangeClipboard !== null`이고 `pastePreview.anchorCol !== null`이면 preview를 표시할 수 있다.
- preview는 score JSON에 저장하지 않는다.
- preview는 paste command가 적용될 시작 column을 보여주는 UI 상태이다.
- preview는 row 위치를 바꾸지 않는다. row 위치는 clipboard의 원본 `sourceRowIds`를 따른다.

## 4. Pointer gesture

### 4.1. 시작 조건

Ctrl + left pointer drag는 영역 선택으로 해석한다.

```text
edit mode
+ left pointer button
+ event.ctrlKey === true
+ hit rowKind is note or global
-> start range selection drag
```

이 조건을 만족하면 기존 note/global rawText drag edit을 시작하지 않는다.
즉 Ctrl + drag는 score mutation을 즉시 수행하지 않는다.

오른쪽 pointer button은 기존 삭제 drag 의미를 유지한다.
Ctrl + right drag는 첫 구현에서 지원하지 않고 일반 right drag 삭제 또는 blocked gesture 중 하나로 처리한다.
권장 정책은 blocked gesture로 두고 status message를 표시하는 것이다.

### 4.2. drag 중 갱신

pointermove마다 현재 hit을 기준으로 임시 range selection을 계산한다.

```text
anchor hit
+ current hit
-> normalize columns
-> collect rowIds between anchor/current in renderer row order
-> keep only same rowKind rows
-> update rangeSelection
-> redraw selection overlay
```

drag 시작 rowKind와 다른 rowKind로 넘어가면 다음 정책을 사용한다.

- note로 시작한 selection은 note row만 포함한다.
- global로 시작한 selection은 global row만 포함한다.
- gap row 또는 canvas 밖 hit은 마지막 유효 range를 유지한다.

이 정책은 `2.8`의 note/global invalidation 분리와 맞추기 위한 것이다.
mixed range selection은 첫 구현에서 만들지 않는다.

### 4.3. pointerup / cancel

pointerup 시 현재 range selection을 확정한다.
pointercancel 시 drag 상태만 종료하고 마지막 range selection을 유지한다.

선택 범위가 1 cell뿐이어도 range selection으로 유지할 수 있다.
다만 단일 click과 시각적으로 헷갈리면 1 cell range는 `selection`으로 축소해도 된다.
첫 구현 권장 정책은 단순성을 위해 1 cell range도 range selection으로 유지하는 것이다.

## 5. Renderer / Overlay

Range selection은 기존 analyzer-derived render item에 섞지 않는다.

권장 구현:

- renderer 또는 app layer에 selection overlay draw 함수를 추가한다.
- overlay는 현재 viewport 기준으로만 그린다.
- overlay canvas 또는 기존 top overlay layer를 사용한다.
- selection overlay는 note/global marker나 note body를 다시 만들지 않는다.

표시 정책:

- 실제 bulk 대상 row 계산에서는 gap row를 제외하더라도, 화면에는 선택된 note/global row 범위를 감싸는 하나의 큰 사각형을 그린다.
- 선택 영역 외곽선을 한 번 그린다.
- active track이 여러 개여도 note row의 cell 영역은 한 번만 표시한다.
- inactive track 표시 alpha와 selection overlay alpha는 서로 독립이다.
- Text off, reverse row, speed scale, zoom, viewport scroll을 모두 반영해야 한다.

Selection overlay redraw 트리거:

- range selection 변경
- scroll
- zoom
- reverse row 변경
- layout apply / preset apply
- score load / Clear All
- range command 이후 selection 유지 또는 해제 정책에 따른 상태 변경

## 6. Paste Preview

### 6.1. 진입과 종료

`Ctrl+C` 또는 `Cmd+C`로 range clipboard가 생성되면 paste preview mode에 들어간다.

```text
range selection
-> Ctrl+C
-> rangeClipboard 생성
-> pastePreview.anchorCol = null
-> score pointer move에서 anchorCol 갱신
```

Paste mode 종료 조건:

- `Ctrl+V` / `Cmd+V` 첫 paste 적용 완료
- `Esc`
- score load
- Clear All
- layout apply / layout preset apply
- range delete
- 단일 click edit
- clipboard가 비워지는 future command

`Ctrl+C` 이후에는 range selection을 해제한다.
clipboard와 paste preview는 유지하되, 영역 선택 모드는 한 번의 복사로 종료한다.
Paste mode가 종료되면 `rangeClipboard`와 `pastePreview`를 함께 비운다.

### 6.2. pointer 추적

clipboard가 있는 상태에서 score 영역 위로 pointer가 움직이면 현재 pointer x 좌표를 column으로 변환해 `pastePreview.anchorCol`에 저장한다.

규칙:

- pointer가 score cell 위에 있으면 해당 hit의 `col`을 사용한다.
- pointer가 score 영역 안에 있지만 row hit이 없으면 x 좌표만으로 column을 계산할 수 있다.
- pointer가 score 영역 밖으로 나가면 preview를 숨긴다.
- preview 숨김은 clipboard 삭제가 아니라 `pastePreview.anchorCol = null`을 뜻한다.
- y 좌표는 preview row 위치에 영향을 주지 않는다.

### 6.3. preview 표시

Paste preview는 selection overlay와 별도 시각 요소로 표시한다.

권장 구현:

- `paste-preview-overlay` DOM layer 또는 canvas overlay를 사용한다.
- preview는 실제 clipboard cell이 있는 위치만 반투명 사각형으로 표시한다.
- row는 clipboard의 원본 `sourceRowIds`를 사용한다.
- column은 `pastePreview.anchorCol + cell.colOffset`을 사용한다.
- score column 범위를 벗어나는 cell은 표시하지 않는다.
- 현재 layout에 없는 rowId 또는 rowKind가 맞지 않는 row는 표시하지 않는다.

시각 정책:

- selection overlay보다 낮은 alpha를 사용한다.
- 예: 청록 계열 반투명 fill과 얇은 점선 또는 실선 outline.
- preview 사각형 높이는 노트 사각형과 같은 높이로 계산한다.
- preview 사각형은 대상 row의 중앙 y 좌표에 맞춰 세로 중앙 정렬한다.
- preview는 note/global marker나 note body를 다시 만들지 않는다.
- scroll / zoom / reverse row / speed scale에 맞춰 위치가 갱신되어야 한다.

### 6.4. paste anchor 우선순위

Paste command의 시작 column은 다음 우선순위를 따른다.

1. `pastePreview.anchorCol`
2. range selection 좌상단 column
3. 단일 selection column
4. 없으면 차단

row 위치는 이 우선순위와 무관하게 clipboard의 원본 `sourceRowIds`를 유지한다.

## 7. Bulk delete

### 7.1. command entry

첫 구현의 삭제 command는 다음 입력을 지원한다.

- `Delete` key
- `Backspace` key
- 우클릭 즉시 삭제는 첫 구현의 primary command로 두지 않는다.
- edit mode의 eraser command와 range selection 조합은 후속 UI 개선으로 둔다.

삭제 command는 `state.rangeSelection !== null`이고 `mode.kind === "edit"`일 때만 동작한다.
view mode에서는 Delete / Backspace가 score를 바꾸지 않는다.

우클릭은 기존 단일 cell 삭제와 충돌할 수 있으므로, range selection이 있을 때 즉시 삭제로 해석하지 않는 것을 권장한다.
후속 구현에서 우클릭을 확장한다면 선택 영역 위 우클릭은 custom context menu를 열고, menu item으로 `Delete`, `Copy`, `Paste`를 제공한다.
context menu가 없는 첫 구현에서는 Delete / Backspace를 range delete의 주 입력으로 둔다.

### 7.2. note range delete

note range delete는 선택된 `trackIds`, `rowIds`, column 범위의 cell을 삭제한다.

```text
rangeSelection(rowKind = note)
-> for each trackId in range.trackIds
-> for each rowId in range.rowIds
-> for col in [startCol, endColExclusive)
-> ScoreTextEdit(rawText = "")
-> existing batch edit path
```

`trackIds`가 비어 있으면 삭제 command를 차단하고 status message를 표시한다.
선택 확정 이후 active track toggle이 바뀐 경우 첫 구현에서는 선택에 저장된 `trackIds`를 사용한다.
이는 사용자가 선택한 당시의 대상 track을 bulk command 대상으로 고정하기 위한 정책이다.

삭제 적용 후에는 range selection을 자동 해제한다.

### 7.3. global range delete

global range delete는 선택된 global rowId / column 범위의 global cell을 삭제한다.

단, col 0 global cell은 기존 edit apply 정책과 같이 삭제할 수 없다.
삭제 command 생성 단계에서 col 0을 제외하거나, edit apply boundary에서 차단된 결과를 status message로 표시한다.

권장 정책:

- col 0은 삭제 edit 목록에 넣지 않는다.
- col 0을 제외하고 삭제된 cell 수를 status message에 표시한다.
- 선택 영역이 col 0만 포함하면 score mutation 없이 warning을 표시한다.

## 8. Copy / Paste

Copy / paste는 paste preview와 함께 구현한다.

### 8.1. clipboard 모델

첫 구현은 app runtime 내부 clipboard를 우선한다.
시스템 clipboard 연동은 후속 선택 사항이다.

```ts
type ScoreRangeClipboard = {
  rowKind: "note" | "global";
  sourceRowIds: RowId[];
  width: number;
  trackIds: TrackId[];
  cells: {
    rowOffset: number;
    colOffset: number;
    trackId?: TrackId;
    rawText: string;
  }[];
};
```

규칙:

- note clipboard는 track별 cell을 보존한다.
- global clipboard는 trackId를 사용하지 않는다.
- 빈 cell은 clipboard에 저장하지 않는다.
- clipboard는 score JSON에 저장하지 않는다.

### 8.2. copy

`Ctrl+C` 또는 `Cmd+C`는 range selection이 있을 때 internal clipboard를 갱신한다.

복사 대상:

- note range: 선택 trackIds / rowIds / col 범위에 실제 존재하는 track cells
- global range: 선택 rowIds / col 범위에 실제 존재하는 global cells

copy는 score mutation을 수행하지 않는다.
copy 이후 paste preview mode에 진입한다.
copy 이후 range selection은 자동 해제된다.

### 8.3. paste

첫 구현 paste는 같은 score layout 안에서만 보장한다.

붙여넣기 anchor:

- paste preview가 보이면 preview column
- preview가 없고 range selection이 있으면 range의 좌상단 column
- preview가 없고 단일 selection이 있으면 해당 cell column
- 마우스가 score cell 위에 있거나 paste 직전 pointer 위치가 있으면 해당 x 좌표의 column을 우선 사용할 수 있다.
- 둘 다 없으면 차단

row mapping:

- clipboard의 원본 `sourceRowIds`를 그대로 대상 rowId로 사용한다.
- 현재 pointer / selection의 x 좌표는 붙여넣기 시작 column 기준으로 사용한다.
- 현재 pointer / selection의 y 좌표는 붙여넣기 row 기준으로 사용하지 않는다.
- 원본 rowId가 현재 layout에 없거나 rowKind가 clipboard rowKind와 다르면 해당 cell은 건너뛴다.
- 대상 col이 score 범위를 벗어나면 해당 cell은 건너뛴다.

note paste의 track 정책:

- 첫 구현은 clipboard에 저장된 trackId를 그대로 사용한다.
- 대상 score에 해당 trackId가 없으면 건너뛴다.
- activeTrackIds로 track을 재해석하지 않는다.

paste는 기존 batch edit 경로를 사용한다.
붙여넣은 cell 수를 status message에 표시한다.
붙여넣기 적용 후에는 range selection을 자동 해제한다.
붙여넣기 적용 후에는 range clipboard와 paste preview도 자동 해제한다.

## 9. 기존 edit 기능과의 관계

Ctrl + drag selection은 기존 edit drag보다 우선한다.

우선순위:

1. busy 상태면 모든 pointer edit 무시
2. view mode loop pick이면 loop pick 처리
3. edit mode + Ctrl + left drag이면 range selection
4. edit mode + right drag이면 기존 delete drag
5. edit mode + left drag이면 기존 rawText drag edit
6. click이면 기존 단일 click / click cycle

range selection이 존재해도 단일 edit click은 계속 허용한다.
단일 edit click이 발생하면 range selection은 해제하는 것을 권장한다.

range selection이 존재한 상태에서 track active toggle이 바뀌면 다음 중 하나를 선택할 수 있다.
첫 구현 정책은 selection을 유지하되, note bulk command 대상은 selection에 저장된 `trackIds`를 사용하는 것이다.
후속 UI에서 혼란이 크면 track toggle 시 range selection을 해제할 수 있다.

Paste preview는 score mutation이 아니므로 parser/analyzer/audio를 갱신하지 않는다.
preview overlay만 다시 그린다.

## 10. Partial rebuild와의 관계

Range selection 자체는 score mutation이 아니므로 parser/analyzer/audio를 갱신하지 않는다.
selection overlay만 다시 그린다.

bulk delete / paste는 기존 `ScoreTextEdit[]` batch edit 경로를 사용한다.
따라서 invalidation kind는 기존 `2.8` 정책을 따른다.

- note range delete / paste -> `noteCell`
- global range delete / paste -> `globalCell`
- mixed range는 첫 구현에서 만들지 않음

대량 range command는 많은 cell을 한 번에 바꿀 수 있으므로 pointermove처럼 frame별로 나누어 적용하지 않는다.
하나의 user command는 하나의 batch edit으로 처리한다.
후속 undo 도입 시에도 하나의 undo group으로 묶기 쉽도록 하기 위함이다.

## 12. 구현 순서

권장 순서:

1. `ScoreRangeSelection` 타입과 range normalize helper 작성
2. Ctrl + drag 상태를 기존 drag edit 상태와 분리
3. selection overlay 첫 구현
4. 단일 click/edit 시 range selection 해제
5. Delete / Backspace bulk delete 구현
6. note/global range delete 단위 테스트
7. copy internal clipboard 구현
8. paste preview overlay 구현
9. paste same-layout 구현
10. browser manual verification

Step 1~4를 `Range Selection MVP`로 보고, Step 5~6을 `Range Delete`, Step 7~9를 `Range Copy/Paste Preview`로 나누어 구현한다.

## 13. 검증 기준

단위 검증:

- Ctrl drag anchor/current hit에서 row/column 범위가 정규화된다.
- note range는 note row만 포함하고 gap/global row를 제외한다.
- global range는 global row만 포함한다.
- note range delete는 선택 당시 `trackIds`를 대상으로 `ScoreTextEdit[]`를 만든다.
- global range delete는 col 0 삭제 edit을 만들지 않는다.
- paste 대상이 score column 범위를 넘으면 해당 cell을 건너뛴다.
- paste는 현재 pointer 또는 selection의 x 좌표를 시작 column으로 사용한다.
- paste는 anchor의 y 좌표와 무관하게 clipboard의 원본 rowId를 보존한다.
- 원본 rowId가 현재 layout에 없거나 clipboard rowKind와 다르면 해당 cell을 건너뛴다.
- paste preview는 clipboard cell의 원본 rowId와 preview anchorCol을 기준으로 표시된다.
- paste preview anchorCol이 null이면 paste preview는 표시되지 않는다.

브라우저 수동 검증:

- edit mode에서 Ctrl + left drag로 영역이 표시된다.
- Ctrl + drag 중 기존 rawText 입력이 발생하지 않는다.
- 일반 left drag edit과 right drag delete는 기존처럼 동작한다.
- 단일 click edit 후 range selection이 해제된다.
- Delete / Backspace로 선택 영역 cell이 삭제된다.
- Delete / Backspace 이후 선택 사각형이 해제된다.
- Ctrl+C 이후 선택 사각형이 해제되고 paste preview만 남는다.
- Ctrl+C 이후 마우스를 score 위에서 움직이면 반투명 paste preview가 따라온다.
- paste preview는 y 좌표가 바뀌어도 원본 row 위치를 유지하고 column만 바뀐다.
- score 영역 밖에서는 paste preview가 숨겨진다.
- Ctrl+V 이후 선택 사각형이 해제된다.
- Ctrl+V 이후 range clipboard와 paste preview가 해제되어 이후 일반 편집 중 preview가 다시 나타나지 않는다.
- Esc로 range clipboard와 paste preview가 해제된다.
- global row col 0은 선택 영역에 있어도 삭제되지 않는다.
- 여러 active track을 켠 상태에서 note range delete가 선택 당시 active track에 적용된다.
- scroll / zoom / reverse row 후 selection overlay 위치가 맞다.
- score load / Clear All / layout apply 후 range selection이 해제된다.
