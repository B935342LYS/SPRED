# 1.43 Practice Playback Render Stabilization

## 1. Scope

노래방/practice mode 테스트 과정에서 발견된 재생, 판정, 렌더링 가독성 문제를 안정화했다.

이번 메모는 다음 변경을 기록한다.

- practice mode beta 표기 및 안내/진단 UI
- pitch input 안정화를 위한 hysteresis와 raw frame 진단
- practice 종료 결과 dialog와 playback 지속 정책
- score per a note / track difficulty 사용자 표시명 정리
- gliss 연결선 렌더 순서 조정
- overlap gain 보정 정책 변경
- cell rawText 길이 상한 완화
- tuplet container edit-only 표시
- dark menu theme의 player/practice 영역 색상 보정

## 2. Practice Mode UI

practice mode 버튼에는 beta 표기를 붙였다.

처음에는 버튼 폭이 부족해 `(beta)`가 잘렸으므로 `practice-mode-button`의 폭과 최소 폭을 늘렸다.

practice panel 상단에는 판정이 엄밀하지 않을 수 있으며 조용한 환경에서 마이크와 헤드셋 사용을 권장한다는 문구를 추가했다.

마이크 입력 문제를 구분하기 위해 진단 줄을 추가했다.

- mic state
- raw frequency Hz
- clarity
- RMS

이 값들은 pitch detection 자체가 입력을 받지 못하는지, 입력은 있으나 confidence/RMS가 낮아 필터링되는지를 구분하는 용도다.

## 3. Pitch Judging

입력 pitch가 target 주변에서 octave 단위로 튀는 문제를 줄이기 위해 hysteresis 기반 보정을 추가했다.

핵심 방향은 다음과 같다.

- 현재 target 후보와 가까운 pitch class인 경우에만 octave correction을 적용한다.
- 이전에 안정적으로 선택된 corrected MIDI를 유지할 수 있으면 유지한다.
- pitch class 차이가 큰 입력은 target으로 억지 보정하지 않는다.

Ok 판정 기준은 60%로 완화했다.

점수 설명 UI에서는 내부 명칭을 사용자 친화적으로 바꿨다.

- `Sample score` -> `score per a note`
- `track weight` -> `track difficulty`

## 4. Practice Finish

playing 상태가 이벤트가 없는 시점에서도 끝나지 않는 문제가 있었다.

현재 정책은 다음과 같다.

- 판정 대상 이벤트가 끝나면 result dialog를 띄운다.
- practice 상태는 finished로 전환한다.
- dialog가 떠도 뒤의 악보 playback은 계속 진행한다.

이는 사용자가 결과를 보면서도 악보 진행을 끊기지 않게 확인할 수 있도록 한 절충안이다.

## 5. Edit UX

`- + start` 조합으로 gliss hold start를 만들 때 기존 absolute pitch / micro pitch modifier가 사라지는 문제가 있었다.

hold start rawText를 합성할 때 기존 pitch modifier suffix를 붙이도록 바꿨다.

예시:

```text
-@g(c,S)@p(60)@m(12.5)
```

## 6. Gliss Render Order

gliss 연결선이 note rectangle 뒤에 있어 잘 보이지 않거나, 반대로 display text를 가리는 문제가 있었다.

렌더 순서를 다음으로 조정했다.

```text
note rectangle/effect
-> gliss connection line
-> display text / mute text / global text
```

이를 위해 gliss marker는 기존 note marker layer에서 제외하고 note layer 중간 단계에 그린다.

부분 갱신에서는 gliss가 있는 경우 dirty 범위 밖 잔상을 막기 위해 note layer 전체 갱신으로 승격한다.

## 7. Overlap Gain

기존 overlap gain은 같은 시간에 울리는 전체 event 수 `N`을 기준으로 각 event에 `1 / N`을 곱했다.

이 방식은 클리핑 방지에는 강하지만 서로 다른 pitch의 화음까지 단음과 비슷한 총 gain으로 눌렀다.

새 정책은 다음과 같다.

```text
same pitch overlap: 1 / duplicate count
different pitch chord: keep gainScale 1
gliss/glissChain: moving pitch로 보고 event별 고유 group
```

즉 같은 pitch가 중복된 경우만 보정하고, 서로 다른 pitch의 화음은 자연스럽게 더 크게 들릴 수 있게 둔다.

## 8. RawText Limit

6잇단/7잇단 head cell에 gliss, absolute pitch, micro pitch modifier가 함께 들어가면 100자 제한을 넘을 수 있다.

cell `rawText` 길이 상한을 100자에서 200자로 올렸다.

문서와 구현 상수도 같은 값으로 맞췄다.

## 9. Tuplet Container Visibility

TupletGroup 점선 컨테이너는 수정 중에는 유용하지만 재생/감상 중에는 note display를 가릴 수 있다.

현재 정책은 다음과 같다.

```text
edit mode: show tuplet containers
view/play/practice mode: hide tuplet containers
```

renderer option에 `showTupletContainers`를 추가하고, app UI sync에서 edit mode일 때만 true를 넘긴다.

## 10. Dark Theme

menu dark theme에서 player/practice 내부 영역 일부가 light color로 남아 있었다.

다음 요소에 dark override를 추가했다.

- player main panel
- practice mode button
- practice panel
- practice rules button
- practice notice
- microphone diagnostic pills
- score/count pills
- practice panel error state

배포본에도 같은 CSS 변경을 반영했다.
