# 1.36. Harmonics Auto Pitch Plan Memo

> 본 메모는 `@p(h)` 특수 토큰 실험을 폐기하고, 기존 `@p(n)` / `@m(c)` 조합으로 하모닉스 sounding pitch를 자동 산출하는 방향을 기록한다.

## 1. 실험 결과

`feature/harmonics-absolute-pitch` 브랜치에서 `@p(h)`를 parser/analyzer/audio/renderer에 통과시키는 실험을 진행했다.

실험 구현은 다음을 포함했다.

- `@p(h)` parser 특수 분기
- `NoteEvent.voicing` 추가
- note/gliss/glissChain의 1~5배음 동시 schedule event 확장
- renderer 마름모 표시
- edit UI `AUTO◇` 옵션

하지만 음악 이론과 구현 비용을 재검토한 결과, 이 접근은 현재 단계에 비해 과하다고 판단했다.

## 2. 폐기 이유

현악기 하모닉스는 보통 1~5배음이 동시에 같은 note처럼 울리는 것이 아니라, 특정 node에서 특정 partial pitch가 주된 sounding pitch로 들리는 주법이다.

따라서 `@p(h)`를 “1~5배음 동시 발음”으로 해석하는 방식은 실제 하모닉스라기보다 additive synth 효과에 가깝다.

또한 마름모 렌더링은 사용자에게 직접 comment 기호 입력을 맡길 수 있으므로, 이를 위해 저장 문법과 analyzer/audio 계약을 넓히는 비용이 크다.

실험 브랜치의 변경은 main에 반영하지 않고 폐기했다.

## 3. 다음 구현 방향

하모닉스는 별도 저장 토큰을 만들지 않고, edit UI의 편의 프리셋으로 구현한다.

기본 방향은 다음과 같다.

- `AUTO◇`는 rawText에 `@p(h)`를 저장하지 않는다.
- 선택된 note row MIDI를 기준으로 하모닉스 sounding pitch를 계산한다.
- 계산 결과는 기존 modifier인 `@p(n)`과 필요한 경우 `@m(c)`로 저장한다.
- renderer와 analyzer는 기존 absolutePitch/microPitch 처리만 사용한다.

이 방식은 parser/analyzer/audio/render 계약을 거의 바꾸지 않고, 사용자가 하모닉스 pitch를 빠르게 입력하게 해준다.

## 4. Partial 후보

현 단계의 기본값은 단일 partial만 발음하는 방향이 적절하다.

후보 값은 다음과 같다.

| Partial | Semitone offset | Stored form example |
| --- | ---: | --- |
| 2nd harmonic | +12.0 | `@p(rowMidi + 12)` |
| 3rd harmonic | +19.02 | `@p(rowMidi + 19)@m(2.0)` |
| 4th harmonic | +24.0 | `@p(rowMidi + 24)` |
| 5th harmonic | +27.86 | `@p(rowMidi + 28)@m(-13.7)` |

초기 구현은 4th harmonic을 기본값으로 두는 것이 단순하다. artificial harmonic에서 자주 기대되는 “두 옥타브 위 sounding pitch”와 대응되며 cent 보정이 필요 없다.

3rd/5th harmonic은 이후 필요하면 드롭다운 옵션으로 확장할 수 있다.

## 5. UI 메모

`AUTO◇`는 absolutePitch dropdown의 특수 선택지 또는 별도 preset으로 둘 수 있다.

저장 시에는 선택된 row MIDI와 partial 설정을 기준으로 `absolutePitch`와 `microPitch` 입력값을 자동 채우거나, rawText 합성 단계에서 바로 `@p(n)` / `@m(c)`를 만든다.

마름모 기호가 필요하면 사용자는 comment/mute cell 또는 custom text로 직접 표시할 수 있다. 하모닉스 기보 자동 렌더링은 현재 우선순위에서 제외한다.
