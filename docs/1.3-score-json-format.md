# 악보 파일 JSON 파일 설계

## 1. 개요

본 문서는 회귀 코드가 사용하는 파일, 파서 중간 산출물, 분석 결과와 이벤트 리스트의 구조, 제약조건, 참조 관계, 상수값을 정의한다.

이 문서의 목적은 구현 전 데이터 구조를 고정하고 추후 구현의 기준으로 삼는 것이다.

## 2. 악보 JSON 파일

### 2.1. 설계 원칙

- 악보 JSON 파일에는 악보 재현에 필요한 최소 데이터만 저장한다.
- 확대율, 스크롤 위치, 텍스트 크기 등 사용자별 화면 상태는 JSON에 저장하지 않는다.
- 셀의 입력값은 파싱된 구조가 아니라 원본 문자열 `rawText` 형태로 저장한다.
- 렌더링, 분석, 오디오 처리를 위해 필요한 중간 구조는 로딩 이후 애플리케이션 내부에서 생성한다.
- JSON은 사람이 읽고 수정할 수 있는 구조를 지향하되, 필요한 제약조건인 참조 무결성과 좌표 유일성은 엄격하게 유지한다.

추가로 본 문서는 저장 구조와 런타임 구조를 명확히 구분하는 것을 전제로 한다.

- 저장 구조는 JSON 파일에 직렬화되는 최소 정보만 다룬다.
- 런타임 인덱스, parser 산출물, analyzer 결과, 렌더링 캐시는 모두 저장 구조와 별개인 실행 중 산출물로 본다.
- 따라서 JSON에는 `parsedCell`, 이벤트 리스트, 렌더 캐시, 오디오 캐시 같은 해석 결과를 저장하지 않는다.
- import/export 경로가 늘어나더라도 저장 포맷 자체는 가능한 한 안정적으로 유지하는 것을 우선한다.

### 2.2. 전체 구조

최상위 구조:

```json
{
  "format": {},
  "musicData": {},
  "instData": {},
  "layout": {},
  "globalLines": {},
  "tracks": []
}
```

### 2.3. 필드 명세: `format`

악보 파일 포맷 자체의 식별 정보이다.

```json
"format": {
  "formatName": "spred-sheet",
  "version": "0.0.1"
}
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `formatName` | string | O | 포맷 이름 | 비어 있지 않아야 한다. 초안에서는 `spred-sheet`를 사용한다. |
| `version` | string | O | 포맷 버전 | 시맨틱 버전 형식 권장 |

### 2.4. 필드 명세: `musicData`

곡과 악보에 대한 메타데이터이다.

```json
"musicData": {
  "musicTitle": "testScore",
  "musicArtist": "testArtist",
  "musicGenre": "testGenre",
  "scoreWriter": "testSheetWriter",
  "comment": "This is a test score for Otamatone DX Basic.",
  "scoreDifficulty": {
      "basic": 1,
      "optional": 5,
      "extra": 10
  },
  "createdAt": "2000-09-09T10:00:00+09:00",
  "updatedAt": "2000-09-09T10:00:00+09:00",
  "youtube": {
    "videoId": "",
    "offsetMs": 0
  }
}
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `musicTitle` | string | O | 곡 제목 | 비어 있지 않음 |
| `musicArtist` | string | O | 작곡가 또는 아티스트 | 문자열 |
| `musicGenre` | string | O | 장르 | 문자열 |
| `scoreWriter` | string | O | 악보 작성자 | 문자열 |
| `comment` | string | O | 작성자의 코멘트 | 100자 제한 |
| `scoreDifficulty` | object | O | 트랙별 난이도 정보 | `basic`, `optional`, `extra`를 모두 가져야 하며 각 값은 정수 권장 |
| `createdAt` | string | O | 생성 날짜 | ISO 형식 사용 |
| `updatedAt` | string | O | 마지막 수정 날짜 | ISO 형식 사용. `createdAt`보다 과거일 수 없음 |
| `youtube.videoId` | string | O | 연동용 유튜브 영상 ID | 빈 문자열 허용 |
| `youtube.offsetMs` | number | O | 유튜브 영상 시작 시점과 악보 시작 시점의 오차 보정용 오프셋(ms) | 정수, 음수 허용 |

`scoreDifficulty` 세부 필드:

추가 규칙:

- `basic`, `optional`, `extra`는 모두 필수이다.
- 새 악보 생성 시 각 값의 기본값은 `0`이다.
- 실제 `tracks`에 아직 셀이 없더라도 난이도 항목은 항상 존재해야 한다.

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `basic` | number | O | 트랙 레이어 basic의 난이도. | 정수 권장, 기본 생성값 0 |
| `optional` | number | O | 트랙 레이어 optional의 난이도. | 정수 권장, 기본 생성값 0 |
| `extra` | number | O | 트랙 레이어 extra의 난이도. | 정수 권장, 기본 생성값 0 |

### 2.5. 필드 명세: `instData`

악기의 종류, 음역대, 현 구성을 정의한다.

추가 규칙:

- `presetId`는 항상 존재해야 한다.
- 기존 프리셋과 다른 사용자 정의 레이아웃 또는 사용자 정의 악기 설정을 저장할 경우 `presetId`는 `"custom"`으로 설정한다.
- `supportsOpen == true`일 경우 모든 현에 `openMidi`가 존재하여야 한다.
- `supportsOpen == false`일 경우 모든 현에 `openMidi`가 존재해서는 안 된다.

```json
"instData": {
  "presetId": "otamatone-basic",
  "family": "otamatone",
  "instName": "Otamatone DX Basic",
  "supportsOpen": false,
  "strings": [
    {
      "stringId": "s1",
      "stringName": "1st String",
      "minMidi": 52,
      "maxMidi": 84
    }
  ]
}
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `presetId` | string | O | 프리셋 식별자 | 문자열. 사용자 정의 레이아웃 또는 사용자 정의 악기 설정은 `"custom"` |
| `family` | string | O | 악기의 분류 | 문자열 |
| `instName` | string | O | 악기 표시명 | 문자열 |
| `supportsOpen` | boolean | O | 개방현 지원 여부 | boolean |
| `strings` | array | O | 현 목록과 각 현의 정보 | 1개 이상 |

`strings[]` 세부 필드:

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `stringId` | string | O | 현 식별자 | 문서 내에서 유일해야 한다 |
| `stringName` | string | O | 현 표시명 | 문자열 |
| `openMidi` | number | X | 개방현을 지원할 경우 개방현의 음정 | 0~127 사이의 정수, `openMidi < minMidi`, `supportsOpen == true`일 경우 필수, `false`면 금지 |
| `minMidi` | number | O | 지판 상의 최저 음정 | 0~127 사이의 정수, `minMidi <= maxMidi` |
| `maxMidi` | number | O | 지판 상의 최고 음정 | 0~127 사이의 정수, `minMidi <= maxMidi` |

### 2.6. 필드 명세: `layout`

행 구조와 높이, 기본 열 폭 등 레이아웃 정보를 정의한다.

추가 규칙:

- 각 `rowDefinitions[].rowId`는 문서 내에서 유일해야 한다.
- `rowDefinitions`의 배열 순서는 `normal` 모드 기준 실제 화면의 행 순서를 의미한다.
- 악보 캔버스의 행을 뒤집어 배치하는 `reverse mode`는 추후 추가 예정이다.
- `type === "gap"`인 행은 편집 불가 행으로 간주한다. 이 값은 별도 `editable` 필드로 저장하지 않고 파생 규칙으로 처리한다.
- `type === "note"`인 행은 동일한 `stringId` 내에서 `midi`가 중복되어서는 안 된다.
- `type === "note"`의 `stringId`는 반드시 `instData.strings[].stringId` 중 하나를 참조해야 한다.
- `type === "gap"`의 `stringId` 역시 반드시 `instData.strings[].stringId` 중 하나를 참조해야 한다.

- rowDefinitions는 instData의 음역대를 참고하여 형성된다. 다만 instData는 악기의 원정보를 나타내며, layout은 사용자의 수정 결과를 반영한 최종 레이아웃이다. 두 정보의 충돌시 layout을 우선한다.

```json
"layout": {
  "baseColumnWidthPx": 21,
  "rowDefinitions": [
    { "rowId": "global-bpm", "type": "global", "kind": "bpm", "height": 21 },
    { "rowId": "global-bpb", "type": "global", "kind": "beatsPerBar", "height": 21 },
    { "rowId": "global-spb", "type": "global", "kind": "stepsPerBeat", "height": 21 },
    { "rowId": "global-dyn", "type": "global", "kind": "dynamics", "height": 21 },
    { "rowId": "s1-gap-51-52", "type": "gap", "stringId": "s1", "fromMidi": 51, "toMidi": 52, "height": 14 },
    { "rowId": "s1-note-52", "type": "note", "stringId": "s1", "midi": 52, "height": 7, "displayLabel": "E3" }
  ]
}
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `baseColumnWidthPx` | number | O | 기본 열 너비(px) | 1 이상의 정수, 기본값 21px |
| `rowDefinitions` | array | O | 문서의 전체 행 정의 | 순서는 화면상 위에서 아래의 순서를 따른다. `normal` 모드 기준 |

`rowDefinitions[]` 공통 필드:

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `rowId` | string | O | 행 식별자 | 문서 내에서 유일해야 한다 |
| `type` | string | O | 행의 종류 | `global`, `note`, `gap` 중 하나 |
| `height` | number | O | 행 높이(px) | 1 이상의 정수 |

행 타입별 추가 필드:

`type: "global"`

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `kind` | string | O | 전역 행의 종류 | `bpm`, `beatsPerBar`, `stepsPerBeat`, `dynamics` 중 하나 |

`type: "note"`

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `stringId` | string | O | 소속 현 ID | `instData.strings[].stringId`를 참조 |
| `midi` | number | O | 해당 행에 할당된 음정의 MIDI note number | 0~127 사이의 정수 |
| `displayLabel` | string | O | 좌측 고정 캔버스에 표시될 행 위 텍스트 | 예: `E3`, `F#3` |

`type: "gap"`

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `stringId` | string | O | 소속 현 ID | `instData.strings[].stringId`를 참조 |
| `fromMidi` | number | O | `normal` 기준 갭행 아래쪽 MIDI | 0~127 사이의 정수 |
| `toMidi` | number | O | `normal` 기준 갭행 위쪽 MIDI | 0~127 사이의 정수 |

### 2.7. 필드 명세: `globalLines`

악보의 현마다 할당되는 악보, 트랙 레이어와 관계없이 전체에 적용되는 전역 셀을 정의한다.

전역 행도 note 행과 같은 원칙으로, 저장 구조에는 해석 결과가 아니라 셀 문자열 자체를 유지한다.
즉 JSON에는 전역 행 셀의 `rowId`, `col`, `rawText`만 저장하고, 숫자값과 ramp 의미는 런타임에서 parser가 `ParsedGlobalCell`로 해석하며, 실제 시간 세그먼트는 analyzer가 생성한다.

추가 규칙:

- `globalLines.cells[].rowId`는 앞서 정의된 `layout.rowDefinitions`를 참조한다.
- `cells`는 로딩 시 `col` 오름차순, 같은 `col` 안에서는 `rowId` 오름차순 정렬이 가능해야 한다.
- 같은 `(rowId, col)` 좌표를 가진 전역 셀은 여러 개 존재할 수 없다.
- 각 전역 kind(`bpm`, `beatsPerBar`, `stepsPerBeat`, `dynamics`)는 `col = 0`에 유효한 시작 셀을 하나 가져야 한다.

```json
"globalLines": {
  "columnCount": 4000,
  "cells": [
      { "rowId": "global-bpm", "col": 0, "rawText": "120" },
      { "rowId": "global-bpb", "col": 0, "rawText": "4" },
      { "rowId": "global-spb", "col": 0, "rawText": "4" },
      { "rowId": "global-dyn", "col": 0, "rawText": "100" }
  ]
}
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `columnCount` | number | O | 악보의 총 길이, 즉 전체 열 수 | 1 이상의 정수 |
| `cells` | array | O | 전역 행 셀 목록 | 전역 셀 배열 |

`cells[]` 세부 필드:

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `rowId` | string | O | 입력된 전역 행의 ID | `layout.rowDefinitions[].rowId`를 참조하며, 해당 row는 `type === "global"`이어야 함 |
| `col` | number | O | 입력된 열 인덱스 | 정수, 범위는 `0 <= col <= columnCount - 1` |
| `rawText` | string | O | 사용자가 입력한 원본 문자열 | 빈 문자열 금지, 해석은 parser가 수행 |

### 2.8. 필드 명세: `tracks`

실제 트랙 레이어별 셀 입력 데이터를 저장하는 목록이다.

추가 규칙:

- 각 `tracks[].trackId`는 문서 내에서 유일해야 한다.
- 같은 트랙 내에서 `(rowId, col)` 좌표는 유일해야 한다.
- `tracks[].cells[].rowId`는 반드시 `layout.rowDefinitions[].rowId` 중 하나를 참조해야 한다.
- `type === "gap"`인 행 ID는 `cells[].rowId`로 참조되어서는 안 된다.
- `trackName`은 단순 표시용 이름이며, 트랙 레이어 분기 구분 로직에는 `trackId`를 이용한다.
- `cells`는 저장 시 `col` 오름차순, 같은 `col` 안에서는 `rowId` 오름차순 정렬을 권장한다.
- 빈 문자열 셀은 저장하지 않는다.

```json
"tracks": [
  {
    "trackId": "basic",
    "trackName": "basic",
    "cells": [
      { "rowId": "s1-note-52", "col": 100, "rawText": "E3" }
    ]
  }
]
```

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `trackId` | string | O | 트랙 식별자 | 문서 내에서 유일해야 한다. 현재 버전에서 허용값은 `basic`, `optional`, `extra` 셋으로 제한한다. |
| `trackName` | string | O | 트랙 표시명 | 표시되는 문자열. 기본값은 `trackId`와 동일 |
| `cells` | array | O | 비어 있지 않은 셀 입력 목록 | 빈 셀은 저장하지 않음 |

`cells[]` 세부 필드:

| 필드 | 타입 | 필수 여부 | 설명 | 제약조건 |
|---|---|---|---|---|
| `rowId` | string | O | 입력된 행의 ID | `layout.rowDefinitions[].rowId`를 참조 |
| `col` | number | O | 입력된 열 인덱스 | 0 이상 `globalLines.columnCount` 미만 정수 |
| `rawText` | string | O | 셀에 입력된 문자열 원본 | 빈 문자열을 저장하지 않음 |

### 2.9. 추가 규칙

아래 규칙은 JSON보다 상위에서 계산되는 규칙이다.

- 행 편집 가능 여부: `row.type == "gap"`이면 편집 불가, 그 외의 행은 편집 가능
- `global` 행 입력 규칙: 숫자와 선형 변화 토큰만 허용한다
- `note` 행 입력 규칙: 노트 문자열 명세를 따르는 `rawText`만 허용한다
- 빈 셀은 JSON에 저장하지 않는다
- 사용자 뷰 상태인 확대 비율, 스크롤 위치, 텍스트 크기, 패널 열림 등의 시각 설정은 JSON에 저장하지 않는다

### 2.10. 검증 시 잡아야 할 오류 목록

- 포맷 정보 누락
- `scoreDifficulty.basic`, `scoreDifficulty.optional`, `scoreDifficulty.extra` 중 하나가 누락된 경우
- `presetId` 누락
- `stringId` 중복
- `rowId` 중복
- `trackId` 중복
- 같은 트랙 내 좌표 중복
- 존재하지 않는 `rowId` 참조
- 존재하지 않는 `stringId` 참조
- `gap` 행에 셀 데이터가 저장된 경우
- `col`이 전체 열 수 범위를 초과한 경우
- 같은 `stringId` 안에서 여러 `note` 행의 `midi`가 중복된 경우
