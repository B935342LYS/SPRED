# 1.24. Step5 전역 Timeline 시각화 구현 메모

## 1. 목적

이 메모는 audio dynamics automation이 연결된 뒤 전역 timeline 시각화를 renderer에 반영하기 위한 1차 규칙을 기록한다.

이 문서는 active renderer/analyzer spec을 보완한다.
Parser, analyzer, score 저장 계약은 이 메모에서 변경하지 않는다.

## 2. 입력 경계

Renderer 변환은 analyzer output만 소비한다.

```text
AnalysisResult.timingTimeline
AnalysisResult.dynamicsTimeline
-> CanvasMarkerItem
-> marker layer draw
```

Renderer는 전역 행 `rawText`를 다시 parse하지 않는다.
전역 rawText 표시는 별도 `CanvasGlobalTextRenderItem` 경로에 남긴다.

## 3. BPM 변경 Marker

BPM 변경 marker는 score 영역 위에 세로선으로 표시한다.

- 초기 `col = 0` BPM 상태는 변경 marker로 그리지 않는다.
- 순간 BPM 변경은 반투명 하늘색 세로선으로 그린다.
- Linear BPM ramp는 ramp 시작점에 marker를 그린다.
- Linear BPM ramp 종료점도 반투명 하늘색 세로선으로 그린다.
- 빨라지는 BPM ramp는 빨간색으로 표시한다.
- 느려지는 BPM ramp는 초록색으로 표시한다.
- `beatsPerBar`, `stepsPerBeat` 변경은 BPM 변경 marker를 만들지 않는다.
- Linear BPM ramp가 beat/step segment boundary로 나뉘어도, 이어지는 ramp의 첫 segment에만 marker를 그린다.
- 같은 tick에는 BPM marker를 하나만 그린다.
- Ramp 종료점과 새 ramp 시작점이 같은 column에 있으면 새 ramp 시작 방향 marker가 하늘색 ramp 종료 marker보다 우선한다.
- `><`가 이전 ramp와 같은 방향의 새 ramp를 시작하더라도 해당 column에 방향 marker를 하나 그린다.

## 4. Dynamics 가이드

Dynamics guide item은 dynamics global row 위에 그린다.

- 값 범위는 active global parser 기준인 `0..150`을 따른다.
- `150`은 dynamics row 전체 높이를 채운다.
- `0`은 보이지 않는다.
- 두께는 row height에 비례한다.

```text
thickness = rowHeight * (value / 150)
```

- 순간 segment는 일정 두께의 밝은 회색 가로 band로 그린다.
- Linear segment는 시작/끝 두께가 보간된 dynamics 값에 맞는 사다리꼴로 그린다.
- Band는 dynamics row 안에서 세로 중앙에 배치한다.
- Global rawText는 기존 global text layer를 통해 guide 위에 그린다.

## 5. 색상 정책

초기 색상은 다음과 같다.

```text
BPM instant: rgba(96, 200, 255, 0.45)
BPM accel:   rgba(255, 80, 80, 0.45)
BPM rit:     rgba(80, 220, 130, 0.45)
Dynamics:    rgba(220, 220, 220, 0.45)
```

## 6. 구현 메모

- `CanvasMarkerItem`에 global timeline marker variant를 추가한다.
- BPM marker는 `timingTimeline`에서 인접 BPM segment의 연속성을 비교해 만든다.
- Dynamics guide marker는 `dynamicsTimeline`과 해당 segment의 source row에서 만든다.
- Dynamics guide는 beat/bar/BPM/gliss marker보다 먼저 그려 text와 foreground marker가 읽히도록 한다.
