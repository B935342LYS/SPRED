# 1.45 Practice Timing Judge Stabilization

## 1. Scope

이번 작업은 practice mode의 pitch correction 제거 이후 안정화 상태를 기준으로, 점수 계산과 timing 판정을 정리한 것이다.

구현 범위:

- target 기반 octave correction, 배음 후보 보정, hysteresis lock 제거 상태 유지
- pitch overlay는 detector가 반환한 생 pitch만 표시
- scoring은 생 pitch를 target pitch class와 직접 비교
- `Miss` sample을 평균 정확도 분모에 포함
- timing onset 기반 `early` / `late` / timing miss 판정 추가
- timing miss는 별도 label이 아니라 최종 pitch label을 `Miss`로 강등
- practice 안내 문구에서 헤드셋 권장 제거
- 판정 overlay 글자 크기와 위치 조정
- 배포 staging 반영

## 2. Pitch Correction Reset

현재 practice pitch 흐름은 다음처럼 단순화되어 있다.

```text
pitchy frequencyHz
  -> frequencyToMidiPitch()
  -> raw midi + centOffset
  -> pitch dot display
  -> pitch class error scoring
```

제거한 항목:

- target 후보를 기준으로 pitch dot을 target 옥타브 주변으로 접는 보정
- target 기반 scoring pitch correction
- 배음 후보를 target 쪽으로 뒤집는 보정
- 이전 안정 pitch를 유지하는 hysteresis lock

이 결정의 이유:

- 보정이 강하면 detector 문제와 판정 문제를 구분하기 어려워진다.
- 사용자가 실제로 낸 pitch가 아니라 target에 가까운 값이 판정에 들어갈 수 있다.
- 보정 제거 후 사용자 수동 테스트에서 pitch 인식 튐이 재발하지 않았으므로, 우선 생 pitch 기반 경로를 유지한다.

남은 주의점:

- 안정화 원인은 완전히 확정되지 않았다.
- 마이크, 연결된 재생 장치, OS/브라우저 입력 처리, echo cancellation / noise suppression 상태가 pitch detector 입력에 영향을 줄 수 있다.
- 따라서 practice UI는 헤드셋을 직접 권장하지 않고, 마이크와 조용한 환경만 권장한다.

## 3. Accuracy And Miss

기존 평균 정확도는 `Perfect`, `Ok`, `Bad` 같은 hit sample 중심으로 계산되어 `Miss`의 `0%`가 전체 평균을 낮추지 못했다.

현재 정책:

```text
accuracyAverage =
  sum(sample.pitchAccuracy for Perfect/Ok/Bad/Miss)
  / count(Perfect/Ok/Bad/Miss)
```

결과:

- `Perfect` + `Miss`는 평균 `50.0%`가 된다.
- 무음, 불인식, clarity/RMS 부족처럼 sample이 생성되지 않은 경우는 평균 분모에 넣지 않는다.
- `Miss`는 음수 감점 없이 scoreContribution `0`이지만, 평균 정확도에는 `0%` sample로 반영된다.

## 4. Timing Judge

Timing 판정은 큰 별도 label을 만들지 않고 기존 pitch 판정을 보조한다.

onset 후보:

- `unvoiced -> voiced` 전환
- voiced 상태에서 pitch class가 충분히 바뀐 순간

판정 기준:

| Timing offset | Overlay | Final label |
| --- | --- | --- |
| `<80ms` | none | pitch label 유지 |
| `80-249ms` | small `early` / `late` | pitch label 유지 |
| `>=250ms` | none | `Miss`로 강등 |

구현 정책:

- `Sync`가 적용된 score time으로 target lookup과 timing 비교를 수행한다.
- 각 `NoteEvent`는 timing 판정을 최대 한 번만 받는다.
- 한 onset 후보는 여러 note에 중복 배정하지 않는다.
- 같은 pitch의 아주 짧은 연속 note는 onset 분리가 불명확하므로 무리하게 timing miss로 강등하지 않는다.

`Miss` 의미:

- timing miss는 timing 전용 label이 아니다.
- pitch가 `Perfect`여도 timing offset이 `250ms` 이상이면 최종 sample은 `Miss`, accuracy `0%`, score `0`이 된다.

## 5. Overlay UI

판정 overlay는 note 근처에 짧게 표시되는 DOM text이다.

현재 크기:

- combo: `15px`
- label: `21px`
- timing: `12px`

배치:

- line-height 계산은 combo / label / timing 줄 수에 맞춰 동적으로 한다.
- 작은 글자 크기에 맞춰 label box 높이를 줄였다.
- 전체 표시 위치를 기존보다 `6px` 위로 올렸다.
- combo는 판정 색을 상속하지 않고 항상 흰색으로 표시한다.

표시 정책:

- `Perfect` / `Ok`는 `COMBO n`을 함께 표시한다.
- `Bad` / `Miss`는 combo 줄을 표시하지 않는다.
- `early` / `late`는 작은 보조 텍스트로만 표시한다.
- timing miss는 `early` / `late` 텍스트 없이 최종 `Miss`로 표시한다.
