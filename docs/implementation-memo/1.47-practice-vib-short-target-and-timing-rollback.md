# 1.47 Practice Vib Short Target And Timing Rollback

## 1. Scope

이번 작업은 practice mode의 vib effect bonus가 짧은 구간에서 잘 인식되지 않는 문제를 보정하고, 이후 시도한 no-onset timing 강등 보정을 롤백한 기록이다.

구현에 남은 변경:

- `GameVibBonusTarget` 길이가 짧을 때 별도 완화 조건 적용
- 160 BPM 기준 2 tick, 약 `187.5ms` vib target 회귀 테스트 추가

시도 후 롤백한 변경:

- 새 timing onset이 없는 긴 note를 `Bad` / `Miss`로 강등하는 보정
- gliss destination note와 gliss 직후 같은 pitch 이웃 note의 no-onset 예외
- 위 보정을 검증하던 관련 테스트

## 2. Vib Short Target

기존 vib bonus 판정은 전체 구간에 공통으로 다음 조건을 요구했다.

```text
VIB_MIN_WINDOW_SECONDS = 0.3
VIB_MIN_FRAME_COUNT = 4
VIB_MIN_DIRECTION_CHANGES = 2
```

160 BPM, `stepsPerBeat = 4` 기준으로 1 tick은 약 `93.75ms`이고 2 tick은 약 `187.5ms`이다.
따라서 2 tick 또는 3 tick 정도의 짧은 vib 구간은 실제로 흔들림이 있어도 `0.3s` 최소 window 조건을 통과하기 어려웠다.

현재 보정:

- target 길이가 `0.25s` 이하이면 short target으로 본다.
- short target의 최소 window는 `max(0.1s, targetDuration * 0.55)`로 완화한다.
- short target의 최소 voiced frame 수는 `3`으로 완화한다.
- short target의 최소 방향 전환 수는 `1`로 완화한다.
- 평균 pitch 근접성, in-tune ratio, amplitude, rate 조건은 유지한다.

이 보정은 아주 짧은 vib를 인정하되, 단순한 flat pitch가 `Vib!`로 오인되는 것을 막기 위해 pitch 중심과 흔들림 조건은 그대로 남긴다.

## 3. Timing Rollback

한참 일찍부터 누르고 있어도 note 시작 시점에서 pitch만 맞으면 `Perfect` overlay가 뜨는 문제가 있었다.
이를 막기 위해 새 onset 후보가 없는 긴 note를 일정 시간 뒤 `Bad` 또는 `Miss`로 강등하는 보정을 시도했다.

문제:

- gliss 끝 note와 뒤쪽 long/vib note가 이웃한 구조에서 정상적으로 legato로 이어 연주해도 뒤 note에서 `Bad` 또는 `Miss`가 발생했다.
- gliss destination 자체, gliss 직후 같은 pitch note 예외를 추가했지만 실제 연주 감각상 여전히 엄격했다.
- 현재 onset detector는 `unvoiced -> voiced` 또는 충분한 pitch-class 변화에 의존하므로, legato/gliss/long/vib 문맥을 단순한 no-onset 벌점으로 다루기 어렵다.

결론:

- no-onset timing 강등 보정은 롤백했다.
- 기존 timing 판정은 onset 후보가 있을 때만 note start와 비교한다.
- onset 후보가 없으면 timing 표시/강등을 만들지 않는 기존 완화 정책으로 돌아갔다.
- 향후 timing 꼼수 방지는 no-onset 자체를 벌점으로 삼기보다 overlay 표시, scoring sample frequency, note 시작부 전용 판정 window, 또는 별도 sustain/attack 판정으로 나누어 재설계한다.
