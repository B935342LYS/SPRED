# 1.44 Practice Pitch Correction Reset

## 배경

노래방 / practice mode의 음정 인식 안정화를 위해 target 기반 octave 보정, 배음 후보 보정, hysteresis 기반 잠금 보정을 실험했다.

실험 과정에서 빠른 음정 이동, 반복 note, 같은 음을 길게 유지하는 상황에서 판정 부작용이 확인되었다.

## 확인한 문제

- target을 기준으로 입력 pitch를 보정하면 사용자가 실제로 낸 음이 아니라 악보 target 쪽으로 이동한 값이 판정에 사용될 수 있다.
- 배음 보정이 특정 상황에서는 `Miss`가 되어야 할 입력을 `Perfect` 또는 `Ok`에 가깝게 만들 수 있다.
- 같은 음을 계속 연주하는 경우에도 새 note가 맞은 것처럼 보이는 문제가 생길 수 있다.
- 보정 함수가 늘어날수록 음정 인식 문제와 판정 문제를 구분하기 어려워졌다.

## 현재 결정

- `78a52d3 홀드 판정 완화` 커밋 상태로 판정 관련 작업을 되돌렸다.
- 당분간 target 기반 pitch correction을 판정 로직에 추가하지 않는다.
- 음정 인식은 우선 detector가 반환하는 생 입력 pitch를 기준으로 다시 검토한다.
- 보정으로 입력을 맞게 만드는 방식보다, 처음부터 정확한 pitch를 안정적으로 잡는 방법을 찾는 방향으로 전환한다.

## 후속 검토 방향

- pitch detector 자체의 한계와 설정값을 다시 점검한다.
- 배음을 target 쪽으로 접어 맞추는 방식은 보류한다.
- 필요하면 CREPE, aubio 계열 YIN/comb 방식 등 공개 pitch detection 구현을 비교 실험한다.
- 보정을 다시 도입하더라도 점수 판정 전에 입력 pitch를 target으로 이동시키는 방식은 피한다.
