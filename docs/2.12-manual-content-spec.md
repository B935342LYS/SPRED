# 2.12. Manual Content 명세

## 1. 문서 목적

본 문서는 `회귀 코드`의 Help > Manual에서 제공할 사용자 매뉴얼의 내용 구조, 작성 양식, 보강 순서, 검수 기준을 정리한다.

`docs/2.11-example-score-and-manual-support-spec.md`가 Manual 진입점과 배포 방식을 다룬다면, 본 문서는 실제 사용자가 읽을 Manual의 목차와 각 항목에 들어갈 설명 범위를 다룬다.

## 2. 기본 원칙

- Manual은 처음 앱을 접한 사용자가 예제 악보를 불러와 재생하고, 기본 편집을 시도할 수 있게 하는 것을 우선한다.
- 기능 설명은 화면의 실제 메뉴명, 버튼명, dialog 이름을 기준으로 작성한다.
- 긴 내부 구현 설명은 Manual에 넣지 않고, 필요한 경우 개발 문서 또는 구현 메모로 분리한다.
- Manual은 정적 HTML 문서로 시작하고, 영상 또는 외부 링크는 보조 자료로 둔다.
- Manual은 언어별/악기별 독립 HTML 문서를 우선한다.
- 공통 내용이 일부 중복되더라도, 악기별 기능 분화가 반복될 때 각 문서를 독립적으로 고칠 수 있는 구조를 우선한다.
- 화면 캡처, 짧은 GIF, YouTube 영상 링크는 추가 가능하지만, 텍스트 설명만으로도 핵심 사용 흐름을 이해할 수 있어야 한다.

## 2.1. 언어 / 악기별 문서 분리 정책

Manual은 단일 HTML 안에서 모든 언어와 악기별 section을 숨김/표시하는 방식보다, 언어와 악기 조합별 HTML을 분리하는 방식을 우선한다.

권장 배포 구조:

```text
regression-code/public/manual/
  index.html
  ko/
    otamatone.html
  en/
    otamatone.html
  assets/
    images/
    styles/
```

`index.html`의 역할:
- Manual 기본 진입점
- language selector 제공
- instrument selector 제공
- 선택한 언어/악기 조합의 HTML로 이동
- 기본값은 첫 구현에서 `ko/otamatone.html`로 둔다.

언어별 문서:
- 한국어 문서를 먼저 작성한다.
- 영어 문서는 한국어 문서가 안정된 뒤 추가한다.
- 언어별 HTML은 서로 다른 파일로 관리한다.

악기별 문서:
- 오타마톤 문서를 첫 악기 문서로 둔다.
- 다른 악기가 추가되면 동일 언어 폴더 아래 별도 HTML로 추가한다.
- 악기별 기능, layout preset, 연주 팁, 주의사항이 달라질 수 있으므로 전체 문서 분리를 허용한다.

중복 정책:
- 첫 구현에서는 공통 섹션 중복을 허용한다.
- 중복이 유지보수 부담이 될 정도로 커지면 build script, template, partial include 도입을 재검토한다.
- 단, 첫 Manual 보강 단계에서는 template/build pipeline을 새로 도입하지 않는다.

## 3. 대상 사용자

### 3.1. 첫 사용자

- SPRED를 처음 열어보는 사용자
- 예제 악보를 불러와 재생해보려는 사용자
- 악보 파일 구조나 편집 규칙을 모르는 사용자

### 3.2. 악보 작성 사용자

- JSON Load / Local Save / JSON Download를 사용하는 사용자
- note cell, global cell, track, layout을 직접 편집하는 사용자
- 기존 악보를 고치거나 새 악보를 만들려는 사용자

### 3.3. 테스트 참여자

- GitHub Pages 배포판에서 예제 DB를 이용해 테스트하는 사용자
- 오류 상황, 재현 절차, 피드백 항목을 기록해야 하는 사용자

## 4. Manual 전체 목차 초안

아래 목차는 첫 보강안의 기본 구조이다.
실제 문서 작성 중 중복되거나 과한 항목은 병합할 수 있다.

```text
Manual
  1. 시작하기
  2. 화면 구성
  3. 예제 악보 불러오기
  4. 재생과 탐색
  5. 파일 저장과 불러오기
  6. 편집 모드 기본
  7. Track 사용
  8. Layout 사용
  9. YouTube 연동
  10. Practice Mode
  11. 검색 / 정렬 / 예제 DB 사용 팁
  12. 오류와 문제 해결
  13. 테스트 피드백 작성 방법
  14. 용어집
```

## 5. 장별 작성 초안

### 5.1. 시작하기

목표:
- 앱을 여는 방법
- 브라우저 권장 환경
- 첫 화면에서 바로 할 수 있는 일
- 예제 악보 또는 로컬 파일로 시작하는 두 경로

작성 후보:
- GitHub Pages 배포판 접속
- 기본 template 자동 로드
- File > Examples
- File > JSON Load
- Help > Manual

### 5.2. 화면 구성

목표:
- 메뉴 영역, 플레이어 영역, 악보 영역, 상태 표시줄을 구분해 설명한다.

작성 후보:
- File / View / Loop / Help 메뉴
- Track, Expand, Clear All
- 중앙 playback controls
- 오른쪽 YouTube panel
- 왼쪽 row label 영역
- 하단 status line

### 5.3. 예제 악보 불러오기

목표:
- Supabase Examples dialog 사용법을 안내한다.

작성 후보:
- File > Examples 열기
- access word 입력
- Load List
- Name / Genre 검색
- Sort / Order 선택
- Load Example
- 현재 악보 교체 확인

주의:
- 예제 목록이 안 뜨는 경우 access word, 네트워크, Edge Function 상태를 확인하도록 안내한다.

### 5.4. 재생과 탐색

목표:
- 악보를 재생하고 원하는 위치로 이동하는 기본 사용법을 설명한다.

작성 후보:
- Play / Stop
- seek bar
- BPM 표시
- Volume
- Wave
- 가로/세로 스크롤
- Fit Height
- Fullscreen
- Speed

### 5.5. 파일 저장과 불러오기

목표:
- 로컬 저장, JSON 다운로드, JSON 불러오기 차이를 설명한다.

작성 후보:
- JSON Load
- JSON Download
- Local Save
- Local Load
- 브라우저 localStorage 사용 시 주의점
- 파일을 다른 사람에게 전달할 때는 JSON Download를 사용한다는 안내

### 5.6. 편집 모드 기본

목표:
- Edit Mode를 켜고 cell을 선택/수정하는 기본 흐름을 설명한다.

작성 후보:
- Edit Mode toggle
- cell 선택
- note 입력
- global 입력
- Undo / Redo
- range selection
- paste preview
- Clear All

주의:
- 모바일은 초기 범위에서 view/load/play 중심이며 편집은 desktop 권장으로 안내한다.

### 5.7. Track 사용

목표:
- basic / optional / extra track의 의미와 표시/편집/재생 관계를 설명한다.

작성 후보:
- Track 메뉴
- active track
- inactive track 표시
- track별 난이도
- Examples 목록의 Difficulty 표시

### 5.8. Layout 사용

목표:
- 악기 preset, row 구조, layout 편집의 목적을 설명한다.

작성 후보:
- layout preset 선택
- layout modify dialog
- string row / pitch row / gap row
- row height
- apply 시 구조 충돌 가능성

### 5.9. YouTube 연동

목표:
- YouTube mode와 앱 playback의 관계를 설명한다.

작성 후보:
- YouTube toggle
- URL or ID
- Reload
- Offset(ms)
- 앱 playback이 YouTube player를 follower로 제어한다는 설명
- 영상 로드 실패 시 기본 악보 재생은 유지된다는 안내

### 5.10. Practice Mode

목표:
- Practice Mode의 목적, 마이크 권한, 판정 방식의 한계를 설명한다.

작성 후보:
- practice mode button
- microphone permission
- Rules
- Sync
- Easy / Std / Pro
- 점수, combo, accuracy
- 결과 dialog
- pitch judging은 보조 기능이며 정확한 음악 평가가 아니라는 안내

### 5.11. 검색 / 정렬 / 예제 DB 사용 팁

목표:
- 예제 DB가 커졌을 때 원하는 악보를 찾는 방법을 설명한다.

작성 후보:
- Name 검색은 제목/아티스트를 함께 대상으로 한다.
- Genre 검색은 부분 검색을 지원한다.
- `Rock` 검색으로 `Indie Rock`, `Pop Rock`, `Jazz Rock`을 함께 찾을 수 있다.
- Difficulty 정렬은 값이 없는 항목을 뒤로 보낸다.
- Updated 정렬은 Ascend/Descend를 바꿔 오래된순/최신순으로 볼 수 있다.

### 5.12. 오류와 문제 해결

목표:
- 사용자가 흔히 마주치는 실패 상황을 짧게 진단할 수 있게 한다.

작성 후보:
- Examples Load List 실패
- Failed to load example score
- JSON Load 실패
- 악보가 비어 보이는 경우
- 소리가 나지 않는 경우
- YouTube 영상이 로드되지 않는 경우
- 마이크 권한이 거부된 경우
- 화면이 너무 작거나 스크롤이 이상한 경우

### 5.13. 테스트 피드백 작성 방법

목표:
- 테스트 참여자가 개발자에게 전달할 정보를 표준화한다.

작성 후보:
- 사용한 브라우저 / OS
- 실행 URL
- 불러온 예제 악보 이름
- 재현 절차
- 기대한 동작
- 실제 동작
- 오류 메시지
- 스크린샷 또는 녹화 첨부 여부

### 5.14. 용어집

목표:
- 앱 안에서 반복되는 용어를 짧게 정의한다.

작성 후보:
- Score JSON
- track
- basic / optional / extra
- cell
- global cell
- note cell
- layout
- preset
- tick
- BPM
- offset
- practice mode
- manifest

## 6. 장별 공통 양식

각 장은 가능한 한 아래 형식을 따른다.

```text
## 장 제목

### 무엇을 하는 기능인가

### 언제 사용하는가

### 사용 절차

### 주의할 점

### 문제가 생겼을 때
```

짧은 기능 설명은 모든 하위 항목을 강제로 채우지 않아도 된다.
다만 사용 절차가 있는 기능은 실제 버튼명과 순서를 포함한다.

## 7. 화면 캡처와 영상 보강 규칙

- 캡처 파일은 Manual 문서와 같은 배포 경로에서 안정적으로 참조할 수 있는 위치에 둔다.
- 캡처는 최신 UI와 맞지 않게 되면 교체하거나 제거한다.
- 영상은 텍스트 Manual을 대체하지 않고 보조 자료로 둔다.
- 영상 링크가 깨져도 Manual의 핵심 절차는 유지되어야 한다.

## 8. 구현 반영 후보

Manual 보강 과정에서 앱 쪽 구현이 추가로 필요할 수 있는 항목이다.

- Manual 내부 목차 anchor
- 현재 앱 버전 또는 문서 갱신일 표시
- Manual index의 language selector
- Manual index의 instrument selector
- 선택값을 저장하는 localStorage 또는 URL parameter
- Examples / Practice Mode 장으로 바로 이동하는 Help submenu
- 오류 메시지별 Manual anchor 연결
- 짧은 FAQ section

## 9. 검수 체크리스트

- 메뉴명과 버튼명이 실제 UI와 일치한다.
- 첫 사용자가 예제 악보를 불러와 재생할 수 있다.
- JSON Load / Download / Local Save / Local Load 차이가 혼동되지 않는다.
- Practice Mode의 한계와 마이크 권한 안내가 포함되어 있다.
- 오류 상황에서 사용자가 현재 악보가 유지되는지, 다시 시도해야 하는지 알 수 있다.
- 모바일 편집 범위가 과장되어 설명되지 않는다.
- 저작권이 불명확한 예제나 영상 사용을 권장하지 않는다.
- `manual/index.html`에서 한국어 오타마톤 Manual로 이동할 수 있다.
- 언어별/악기별 문서 링크가 깨지지 않는다.

## 10. 미정 항목

- 캡처 이미지 저장 위치
- 영상 링크 또는 재생목록 URL
- Manual에 포함할 예제 악보명 목록
- 영어 Manual 추가 시점
- 오타마톤 외 첫 추가 악기 후보
