# 2.7. YouTube Sync UI / Playback 명세

## 1. 문서 목적

본 문서는 `회귀 코드`의 YouTube 반주 동기화 기능을 정의한다.

목표는 다음이다.

- `ScoreFile.musicData.youtube`에 저장된 YouTube 영상 정보를 읽는다.
- YouTube mode를 켜면 앱 우상단 영역에 영상을 표시한다.
- 악보 playback controller의 현재 score time을 기준으로 YouTube 재생 위치를 맞춘다.
- 사용자가 video 입력 또는 offset을 수정하고 `Reload`를 누르면 같은 score 위치에서 YouTube 영상을 다시 읽고 위치를 맞춘다.
- loop playback보다 YouTube sync 구현을 우선한다.

본 문서는 `docs/1.3-score-json-format.md`, `docs/2.0-ui-mvp-spec.md`, `docs/2.3-audio-playback-module-spec.md`, `docs/2.6-track-layer-ui-spec.md`의 후속 구현 명세이다.

## 2. 기본 원칙

YouTube sync는 score 자체의 parser/analyzer 계약을 바꾸지 않는다.

```text
ScoreFile.musicData.youtube
  -> YouTube UI state
  -> YouTube iframe player
  -> PlaybackController score time과 동기화
```

YouTube mode는 accompaniment playback의 첫 외부 영상 구현으로 본다.
회귀 코드의 자체 oscillator audio와 YouTube 영상은 같은 `scoreSeconds`를 기준으로 동작하되, 실제 음원은 서로 다른 backend이다.

첫 구현에서는 YouTube mode가 켜져 있어도 자체 oscillator audio를 강제로 끄지 않는다.
사용자는 기존 volume control로 자체 audio 음량을 조절할 수 있다.
YouTube mode 활성 시 자체 audio 자동 mute 정책은 두지 않는다.

## 3. 저장 형식

저장 형식은 기존 `musicData.youtube`를 사용한다.

```ts
type YoutubeSync = {
  videoId: string;
  offsetMs: number;
};
```

`videoId`는 저장 시 YouTube video id만 보관한다.
사용자 입력 UI는 다음 형태를 모두 받을 수 있다.

- `https://www.youtube.com/watch?v=VIDEO_ID`
- `https://youtu.be/VIDEO_ID`
- `https://www.youtube.com/embed/VIDEO_ID`
- `VIDEO_ID`

입력값은 UI 또는 app 계층에서 `videoId`로 정규화한다.
정규화 실패 시 score 저장값은 바꾸지 않고 상태 메시지로 오류를 표시한다.

`offsetMs`는 정수 millisecond 값이다.
의미는 다음으로 고정한다.

```text
youtubeSeconds = scoreSeconds - offsetMs / 1000
```

예:

- 영상 앞부분에 12.5초의 인트로가 있고 악보 0초가 영상 12.5초와 맞으면 `offsetMs = -12500`
- 영상이 악보보다 0.3초 늦게 시작해야 하면 `offsetMs = 300`

`youtubeSeconds < 0`이면 YouTube player는 0초에 머문다.
이 경우 score time이 `offsetMs / 1000`에 도달하기 전까지는 영상 재생을 시작하지 않거나, 재생 요청 후 즉시 0초에 정렬한다.

부호 의미는 사용자 조작 기준으로 둔다.

- 음수 offset은 YouTube를 더 빠르게 시작하게 한다.
- 양수 offset은 YouTube를 더 느리게 시작하게 한다.

사용자 표시 라벨은 `Offset (ms)`를 사용한다.
내부 필드명과 저장 형식은 계속 `offsetMs`를 사용한다.

## 4. UI 범위

오른쪽 YouTube 영역은 다음 상태를 가진다.

```ts
type YoutubeModeState =
  | { kind: "off" }
  | { kind: "loading"; videoId: string; offsetMs: number }
  | { kind: "ready"; videoId: string; offsetMs: number }
  | { kind: "error"; message: string };
```

첫 구현 UI 요소:

- YouTube mode toggle
- video URL 또는 video id 입력
- offset ms 숫자 입력
- `Reload` 버튼
- YouTube iframe 표시 영역
- 연결 상태 텍스트

입력 초기값:

- video 입력은 현재 `score.musicData.youtube.videoId`
- offset 입력은 현재 `score.musicData.youtube.offsetMs`

YouTube mode off:

- iframe은 숨기거나 placeholder를 표시한다.
- `Reload`는 disabled 처리한다.
- video/offset 입력은 YouTube 영역에서 계속 편집할 수 있다.
- YouTube mode가 off 또는 error여도 video/offset 입력과 `Reload`는 접근 가능하게 둔다.

YouTube mode on:

- `videoId`가 비어 있으면 mode를 켜지 않고 상태 메시지를 표시하며 YouTube mode를 off 상태로 유지한다.
- 유효한 `videoId`가 있으면 iframe player를 생성하거나 기존 player에 새 video를 load한다.
- player ready 후 현재 score time에 대응하는 `youtubeSeconds`로 seek한다.
- 재생 중이면 YouTube도 재생한다.
- paused/stopped 상태이면 YouTube도 pause 상태로 둔다.

## 5. Playback 동기화 정책

YouTube sync는 기존 `PlaybackController`의 score time을 authoritative source로 본다.
YouTube player time은 score time을 따라가는 follower이다.

기본 변환:

```ts
youtubeSeconds = scoreSeconds - offsetMs / 1000;
```

동작별 정책:

| App 동작 | YouTube mode on 동작 |
| --- | --- |
| play from stopped | player ready 확인 후 `youtubeSeconds`로 seek, YouTube play, score playback start |
| resume from paused | 현재 score time 기준 `youtubeSeconds`로 seek 후 YouTube play |
| pause | YouTube pause, score playback pause |
| stop | YouTube pause 후 `scoreSeconds = 0`에 해당하는 위치로 seek |
| seek bar 변경 | score seek 적용 후 YouTube를 새 `youtubeSeconds`로 seek |
| score scroll 기반 seek | seek bar와 같은 정책 |
| score load | 기존 player reset, 새 score의 `musicData.youtube` 값으로 UI 갱신 |
| metadata edit apply | player는 즉시 새 입력을 반영하지 않고, `Reload` 또는 mode toggle에서 적용 |
| active track 변경 | YouTube player는 유지, score time도 유지 |
| layout apply | playback runtime reset과 함께 YouTube는 pause 상태로 정렬 |

첫 구현에서는 drift 보정 주기를 단순하게 둔다.

- play/resume/seek/reload 시 명시적으로 `seekTo()`를 호출한다.
- 재생 중에는 1초 내외의 interval로 score time과 player time 차이를 확인한다.
- 차이가 250ms보다 크면 YouTube를 다시 seek한다.
- 250ms 이내 차이는 YouTube player 자체 재생을 신뢰한다.

250ms는 일반적인 수동 청감에서 싱크 차이를 인지하기 어려운 수준을 목표로 한 첫 구현값이다.
수동 검증에서 차이가 분명히 들리면 더 낮출 수 있다.

## 6. Reload 정책

`Reload` 버튼은 현재 score 위치를 유지한 채 video 입력과 offset 입력을 다시 읽고 YouTube player를 재정렬한다.

동작 순서:

1. YouTube 영역의 video 입력을 `videoId`로 정규화한다.
2. offset 입력을 정수 ms로 읽는다.
3. `score.musicData.youtube`를 새 `{ videoId, offsetMs }`로 갱신한다.
4. 현재 score time을 읽는다.
5. `youtubeSeconds = scoreSeconds - offsetMs / 1000`을 계산한다.
6. 기존 player가 있으면 같은 iframe 영역에서 새 video를 load하거나 player를 재생성한다.
7. player가 준비되어 있으면 해당 위치로 seek한다.
8. 현재 playback state가 playing이면 YouTube를 play한다.
9. stopped 또는 paused이면 YouTube를 pause 상태로 둔다.

`Reload`는 score의 note/global cell, parser result, analyzer result를 바꾸지 않는다.
따라서 runtime rebuild는 필요하지 않다.
다만 score metadata가 바뀌므로 JSON download/local save 대상에는 새 값이 반영되어야 한다.

## 7. YouTube IFrame API

첫 구현은 YouTube IFrame Player API를 사용한다.

예상 파일:

```text
regression-code/src/app/youtube/
  youtube_types.ts
  youtube_url.ts
  youtube_player.ts
  youtube_sync.ts
  youtube_binding.ts
```

역할:

- `youtube_types.ts`
  - YouTube mode state, sync input, player wrapper 타입
- `youtube_url.ts`
  - URL 또는 id 입력에서 `videoId` 추출
- `youtube_player.ts`
  - IFrame API script load, `YT.Player` 생성, load/seek/play/pause/dispose wrapper
- `youtube_sync.ts`
  - `scoreSeconds - offsetMs` 변환, drift check, reload 정책
- `youtube_binding.ts`
  - DOM event binding, UI state sync, app playback runtime과 연결

IFrame API script는 lazy load한다.
즉 YouTube mode가 처음 켜질 때 로드한다.
앱 부팅 시에는 외부 script를 바로 로드하지 않는다.

## 8. 오류 처리

오류는 왼쪽 status line 또는 YouTube 영역 상태 텍스트에 짧게 표시한다.

오류 종류:

- video id가 비어 있음
- URL에서 video id를 추출할 수 없음
- YouTube IFrame API 로드 실패
- player ready timeout
- iframe playback이 브라우저 autoplay 정책으로 차단됨
- 영상 embedding이 허용되지 않음

video id가 비어 있거나 embedding이 허용되지 않는 영상이면 YouTube mode를 off로 전환하고 iframe을 비활성화한다.
상태 메시지에는 원인을 짧게 표시한다.

autoplay 정책 대응:

- 사용자의 play button click 안에서 YouTube `playVideo()`를 호출한다.
- 차단되면 YouTube mode는 유지하되 상태를 error 또는 paused로 표시한다.
- 사용자가 iframe 자체를 눌러 재생할 수 있게 한다.

## 9. 상태와 저장 반영

YouTube 입력값은 오른쪽 YouTube 영역에서 수정한다.

YouTube mode가 off 또는 error 상태여도 오른쪽 YouTube 영역의 video/offset 입력과 `Reload`는 접근 가능하게 둔다.
따라서 Details dialog는 title, artist, genre, writer, comment, difficulty 같은 일반 score metadata만 다룬다.
YouTube video id와 offset은 Details dialog에 두지 않는다.

권장 정책:

- YouTube 영역의 `Reload`는 youtube 필드만 갱신한다.
- `Reload`로 `musicData.youtube`가 바뀌면 `updatedAt`도 갱신한다.
- YouTube mode on/off 상태는 score JSON에 저장하지 않는 runtime UI 상태로 둔다.
- 저장된 video id가 비어 있으면 YouTube mode를 off로 유지한다.

YouTube 영역의 video/offset 입력은 입력 중에는 score metadata와 iframe player에 실시간 반영하지 않는다.
반영 시점은 `Reload`로 제한한다.
빈 video 입력으로 `Reload`하면 저장된 `videoId`를 빈 문자열로 갱신하고 YouTube mode를 off로 전환한다.

## 10. Track Layer / Playback Reset과의 관계

Track layer active 변경은 YouTube player를 재생성하지 않는다.
이유는 YouTube sync가 track 선택과 무관하게 score time만 참조하기 때문이다.

다음 이벤트는 YouTube player를 pause 또는 reset한다.

- score load
- Clear All
- layout apply
- playback stop

다음 이벤트는 YouTube player를 유지하고 위치만 재조정한다.

- play
- pause
- resume
- seek
- active track toggle
- metadata youtube reload

## 11. 검증 기준

타입/단위 검증:

- YouTube URL parser가 watch/youtu.be/embed/id 입력을 `videoId`로 정규화한다.
- 빈 문자열과 잘못된 URL은 실패한다.
- `scoreSeconds - offsetMs / 1000` 변환이 양수/음수 offset 모두에서 기대값을 반환한다.
- `youtubeSeconds < 0`이면 0초로 clamp하는 정책이 적용된다.

수동 브라우저 검증:

- YouTube mode off에서는 iframe이 재생되지 않는다.
- video id가 비어 있으면 mode on이 차단되고 상태 메시지가 표시되며 YouTube mode가 off로 유지된다.
- mode on 후 우상단에 영상이 표시된다.
- play를 누르면 score playback과 YouTube가 함께 시작한다.
- pause를 누르면 둘 다 멈춘다.
- resume은 pause 위치에서 둘 다 이어진다.
- stop은 score 0초와 해당 YouTube offset 위치로 돌아간다.
- seek bar 이동 후 YouTube 위치가 다시 맞는다.
- video 입력 또는 offset 값을 바꾸고 `Reload`를 누르면 현재 score 위치는 유지되고 YouTube 영상/위치가 다시 적용된다.
- embedding이 막힌 영상은 오류 메시지를 표시하고 YouTube mode를 off로 전환한다.
- score load 후 새 score의 `musicData.youtube`가 UI에 반영된다.

권장 명령:

```text
npm run typecheck
npm run test:score
npm run test:audio
npm run build
```

YouTube URL parser와 sync 계산 helper가 추가되면 별도 `test:youtube` 또는 기존 app test에 포함한다.

## 12. 확정된 구현 결정

1. YouTube mode on일 때 자체 oscillator audio는 자동 mute하지 않는다.
   - 사용자가 기존 volume control로 직접 조절한다.
2. 사용자 표시 라벨은 `Offset (ms)`를 사용한다.
   - 첫 구현에서는 `Offset (ms)`를 사용한다.
   - 내부 저장 필드는 `offsetMs`로 유지한다.
3. drift 보정 threshold는 첫 구현에서 250ms로 둔다.
   - 사용자가 명확히 인지하는 싱크 차이가 남으면 후속 조정한다.
4. YouTube 영역의 video/offset 입력은 `Reload` 시점에만 반영한다.
   - 입력 중 실시간 반영은 하지 않는다.
5. video 입력이 없거나 embedding이 막힌 영상은 YouTube mode를 off로 전환한다.
   - 오류 문구를 함께 표시한다.
6. Details dialog에서는 YouTube video/offset 입력을 제거한다.
   - YouTube 관련 입력은 오른쪽 YouTube 패널 내부에 둔다.
7. YouTube mode on/off는 score JSON에 저장하지 않는 runtime 상태로 둔다.
8. YouTube player는 score time을 기준으로 따라가며, 사용자가 iframe에서 직접 seek해도 drift check에서 score time 기준으로 다시 정렬한다.
9. YouTube URL parser는 `watch?v=`, `youtu.be/`, `embed/`, `shorts/`, raw id 입력을 허용한다.
10. YouTube 화면 높이는 우상단 패널에서 최소한으로 유지하고, 메뉴 영역 높이를 늘리지 않도록 한다.
